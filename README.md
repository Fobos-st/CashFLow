CashFlow, сайт для ведения ДДС

К сожалению сроки(я и так не успел) + и в целом свободного времени не хватило дореализовать идейно и структурно весь проект.
Изначально он должен был иметь:
Система регистрации авторизации: Сделана
Счета: Реализованы только персональные, так-же планировались групповые но для них реализована БД с отдельными статусами, транзакциями, категориями и списком участников и форма для создания счета(только фронтенд часть)
Создание новых транзакций: Реализована базовая форма без бизнес логики(Поле выбора категорий, автоматически фильтрующее подкатегории на основе выбранной категории.)
Редактирование тразакций: Не успел
Создание собсвтенных статусов, типов, категорий, подкатегорий: Не успел
Фильтрация истории транзакций: Готова лишь фронтенд часть + js для бизнес логики, что к определённому типу относится определённые категории и к категориям подкатегории

В целом из этого всего на мой взгял не критично по коду это реализация "Редактирования транзакций", "Создание собсвтенных статусов, типов, категорий, подкатегорий" - Это всё в большой мере просто фронтенд и базовые CRUD операции
